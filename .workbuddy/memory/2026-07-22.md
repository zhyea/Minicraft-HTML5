## 背包物品说明功能（已实现，三道门全绿）
- 需求：为背包 InventoryMenu 中每件物品加中文说明。
- 流程：主理人诊断 → 调度 design-strategist 出文案与展示方案（`design/item-descriptions.md`，覆盖 33 条：21 资源 + 5 工具类型 + 6 家具 + 1 力量手套）→ 调度 engineering-lead 落地。
- 数据模型：`Item.getDescription()` 基类默认 `''`；`Resource` 加 `description` 字段（构造第 4 参，默认 `''`），`ResourceItem`/`PlantableResource`/`FoodResource` 透传；`ToolItem` 用 `TOOL_DESC` 表（按 type.name：剑/斧/锄/镐/铲）+ 统一等级后缀；`FurnitureItem` 用 `FURNITURE_DESC` 表（按 furniture.name）；`PowerGloveItem` 常量。
- 展示：`InventoryMenu.vue` 新增 `selectedDesc` computed，列表下方 `<p v-if="selectedDesc" class="item-desc">` 显示，空串自动隐藏（随选中项实时刷新）。
- 测试：`ts-port/src/game/__tests__/item-description.test.ts`（7 测试，installTiles+installResources）。
- 验证门：tsc=0 / vitest 109（含新增 7）/ vite build 成功。未 git commit。
- 注意：design-strategist 指出物品真正基类是 `Item`（Resource 由 ResourceItem 包装）；其 §6「CJK 字库」是 GWT(Java) 8×8 点阵管线，ts-port 的 Vue 覆盖层中文走浏览器原生渲染，无需字库改造。
- 设计侧还纠正过：口述"基类 Resource"实为 Item。

## 背包宫格化 + 物品聚合（已实现，三道门全绿）
- 需求：① 同类物品合并显示数量（如多个木剑→"木剑 xN"而非每行一个）；② 背包改宫格布局。
- 诊断：`Inventory.add` 对 ResourceItem 已自动合并计数，但 ToolItem/FurnitureItem 等每实例独立存，同名多个会各占一行；`ToolItem.getName()`=LEVEL_NAMES[level]+type.name（木剑/石斧），`matches()` 按 type.name+level 区分；`equipItem` 只设 activeItem 不移除，聚合后回传代表实例安全。
- 改动：新增纯函数 `ts-port/src/ui/inventoryGroup.ts` 的 `groupItems(items)` 按 `getName()` 聚合并累加 count（ResourceItem 用 item.count，其余每个实例 +1）；`InventoryMenu.vue` 的 `rows` 改为 `groupItems(props.items)`，模板由 `<ul item-list>` 改为 4 列 `.item-grid`（每格 button.item-cell：ItemIcon + cell-name + cell-count 角标 count>1 显示）；导航 ArrowLeft/Right ±1、ArrowUp/Down ±4（wrap clamp）；`.btn-row`/`.saved-flash`/`.hint`/`.item-desc` 保留；`menus.test.ts` 选择器由 .item-row 更新为 .item-cell。新增 `ts-port/src/game/__tests__/inventory-group.test.ts`（5 测试：同名工具合并/异名分离/资源按count累加/多实例求和/异资源分离）。
- 验证门：tsc=0 / vitest 117（含新增 5，menus.test 同步更新）/ vite build 成功。未 git commit。
- 说明：木剑与木斧是不同类型（剑 vs 斧）本就该分两格；"合并"只针对同名多个（如同合成了两把木剑）。
