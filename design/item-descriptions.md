# Minicraft-HTML5 物品说明文案与展示方案（zh-CN）

> 本文档由设计 / 文案专家（design-strategist）产出，**未改动任何 `.ts` / `.vue` 源码**。
> 用途：为背包中每件物品增加一段简短中文说明（鼠标 / 方向键选中时，提示"这是什么、有什么用"）。
> 物品名沿用既有本地化规范 `design/i18n/zh-CN-strings.md`（下称「i18n 文档」），本文档仅补充**说明文本**与**展示 / 取数方案**。

---

## 1. 前提与约定

1. **类层级纠正（重要）**：口述将基类记为 `Resource`，经核对 TS 端口，真实基类为 **`Item`**
   （`ts-port/src/game/item/Item.ts`）。`Resource` 是**资源定义类**（承载 `name` / `sprite` / `color`），
   由 **`ResourceItem`** 包装（`ResourceItem.resource`）。因此"基类 `getDescription()` 默认返回空串"应落在 `Item` 上。
2. **四个具体子类**均需覆盖 `getDescription()`：`ResourceItem`、`ToolItem`、`FurnitureItem`、`PowerGloveItem`
   （见 §5 边界处理）。它们就是游戏里全部的物品类型，覆盖后即覆盖全部 33 条说明。
3. **字体管线依赖（必须）**：说明文本为中文，依赖 i18n 文档 §5 注意点 1 的「CJK 点阵字库」改造；
   且本方案新增大量 i18n 文档附录 A 之外的汉字，需把本文档字符集并入字库（见 §6）。
4. **交互动词对齐**：说明里的"**按 X 打开 / 使用**"与"**按攻击键（C）**"对齐 i18n 文档 §2.2 的控制键定义。
5. **行宽约束**：背包框约 160px 宽，中文 8px/字（i18n 文档 §1）。故**单行建议 ≤ 18 中文字（≈144px）**，
   整段说明 **20–40 字、折 1–2 行**，避免溢出。

---

## 2. 展示方案建议

### 2.1 位置与时机
在 `InventoryMenu`（目前仅渲染 图标 + 名字 + 数量）的物品列表中，**选中某项时**，于列表**下方**预留一个说明区，
显示 `选中项.getDescription()` 的返回文本。选中项随鼠标悬停或方向键移动而切换，说明区实时刷新。

### 2.2 Mock 布局（渲染层无关，仅示意）

```
┌──────────────────────────┐
│          背包            │   ← renderFrame 标题「背包」（i18n §2.7 已定）
│                          │
│  > 木                x12 │   ← 选中行（光标 > < 保留 ASCII，i18n §2.9）
│    石                 x3 │
│    苹果               x2 │
│    木剑                1 │
│    工作台              1 │
│    …（其余物品）         │
│                          │
│  ──────────────────────  │   ← 分隔线（可选，纯视觉）
│  砍树获得，用途最广的      │   ← getDescription() 文本（选中项）
│  基础材料，可合成工具家具。│      单行 ≤18 中文字，整段 20~40 字折 1~2 行
└──────────────────────────┘
```

### 2.3 实现层提示
- **经典 Screen 渲染**：在 `InventoryMenu.render()` 列表绘制之后，取 `items[selectedIndex].getDescription()`，
  用与名字相同的 CJK `Font.draw` 绘制于列表下方（注意 §1.5 行宽，必要时按 18 字折行）。
- **Vue UI 层**（`ts-port/src/ui/`，如 `ItemIcon.vue` 所在体系）：将选中项的 `getDescription()` 绑定到列表下方一个文本节点即可，
  文本来源同为 `item.getDescription()`，无需改动物品类本身。
- **空串兜底**：基类默认返回 `""`，说明区可据此隐藏分隔线与文本（避免空框）。

## 5. 边界处理（getDescription 落地方案）

> 目标：基类默认空串，四个具体子类各自覆盖。下面给出**推荐取数方式**（仍由 engineering-lead 落地，本文档不改码）。

### 5.1 基类：Item（默认空串）
在 `ts-port/src/game/item/Item.ts` 新增（即口述的"基类默认返回空串"）：
```ts
public getDescription(): string {
  return '';
}
```

### 5.2 ResourceItem —— 需要覆盖
`ResourceItem` 包装 `this.resource: Resource`，而 21 种资源是 **`Resource` 实例**（非子类），
故逐资源文本需落在 `Resource` 上或集中查表。两种推荐：
- **方案 A（推荐，与 `name/sprite/color` 同构）**：给 `Resource`
  （`ts-port/src/game/item/resource/Resource.ts`）加字段 `description: string`，
  在 `registry.ts` 安装 21 个单例时一并赋值；`ResourceItem.getDescription()` 直接 `return this.resource.description;`
- **方案 B（最小改动 Resource）**：在 `ResourceItem` 内维护 `Map<string, string>`（键用 `this.resource.name`，
  `Resource` 已有 `BY_NAME` / `getByName` 机制，可平行再加一张描述表），查表返回。
- §4.1 已给出 21 条文本，直接填入即可。

### 5.3 ToolItem —— 需要覆盖
`ToolItem` 有 `type: ToolType`（`type.name` ∈ {铲,锄,剑,镐,斧}）与 `level`（0–4，对应 `LEVEL_NAMES = ['木','石','铁','金','宝石']`）。
实现：
```ts
public getDescription(): string {
  const base = TOOL_DESC[this.type.name];          // 取 §4.2 类型级文本
  const tier = ToolItem.LEVEL_NAMES[this.level];    // 当前材质名
  return `${base}（${tier}级；木<石<铁<金<宝石，等级越高，伤害或效率越高。）`;
}
```
（等级提示文本可精简，但须保留"等级越高，伤害/效率越高"这一句；只有剑/斧 `canAttack()` 为真，属玩法事实，说明已点明。）

### 5.4 FurnitureItem —— 需要覆盖
`FurnitureItem` 包装 `this.furniture: Furniture`，`getName()` 即 `furniture.name`
（工作台 / 铁砧 / 熔炉 / 烤箱 / 箱子 / 灯笼，见 i18n §4.2）。实现按 `this.furniture.name` 分流返回 §4.3 文本即可。

### 5.5 PowerGloveItem —— 需要覆盖
`PowerGloveItem.getName()` 固定返回「力量手套」。直接返回 §4.4 常量文本，无需分支。

### 5.6 覆盖结论
四个具体子类（`ResourceItem` / `ToolItem` / `FurnitureItem` / `PowerGloveItem`）即游戏全部物品类型；
逐一覆盖后，`Item` 基类空串仅作为未预见类型的兜底，**无需再为其他类添加覆盖**。

## 6. 字库依赖（必须并入工程）

说明文本引入了 i18n 文档附录 A **之外**的大量汉字。请工程以脚本从**本文件**提取全部 CJK 与全角标点，
与 `design/i18n/zh-CN-strings.md` 附录 A 取**并集**后纳入点阵字库（i18n §5 注意点 1、6）。

**本文件新增的代表性汉字（供快速核对，权威以脚本提取为准）：**
```
砍途广础岩制作更耐装备饰世界添抹亮慢填平洞穴积浆自由改造形
漠方块熟收碰触受耕种发芽再烤拿里手持键吃下恢复命值掉落燃料
常见靠近打造脆史莱姆灯笼版本层特殊像被置深处集满顶武必备帮手
快效率提升主要同时界面
```
**新增全角标点**：本文件说明中首次出现 `；`（全角分号）与 `（` `）`（全角括号），须确认字库已含。
